# Create Command - 创建命令

创建新的Claude Code自定义命令，具有适当的结构和最佳实践。

## 使用方法:

`/create-command-zh [command-name] [description]`

## 执行流程:

### 1. 命令分析

- 确定命令目的和范围
- 选择适当位置（项目级 vs 用户级）
- 分析类似的现有命令以寻找模式

### 2. 命令结构规划

- 定义所需参数和参数
- 规划命令工作流程和步骤
- 识别所需工具和权限
- 考虑错误处理和边界情况

### 3. 命令创建

- 使用适当的YAML前言创建命令文件
- 包含全面的文档
- 添加使用示例和参数描述
- 使用`$ARGUMENTS`实现适当的参数处理

### 4. 质量保证

- 验证命令语法和结构
- 测试命令功能
- 确保适当的工具权限
- 根据最佳实践进行审查

## 模板结构:

```markdown
---
description: 命令的简要描述
description-hint: 预期参数格式
allowed-tools: 所需工具列表
---

# 命令名称

此命令的详细描述以及何时使用它。

## 使用方法:

`/[category:]command-name [arguments]`

## 执行流程:

1. 分步说明
2. 清晰的工作流程定义
3. 错误处理考虑

## 示例:

- 具体的使用示例
- 不同的参数组合

## 注意事项:

- 重要考虑因素
- 限制或要求
```

## 最佳实践:

- 保持命令专注且单一目的
- 使用描述性名称和清晰的文档
- 在前言中包含适当的工具权限
- 提供有用的示例和使用模式
- 使用验证优雅地处理参数
- 遵循现有的命令约定
- 在部署前进行彻底测试

## 你的任务:

按照以下指南创建名为"$ARGUMENTS"的新命令：

1. 如果描述不清楚，询问命令目的的澄清
2. 确定适当的位置（项目级 vs 用户级）和类别（例如gh、cc或询问其他类别）
3. 使用适当的结构创建命令文件
4. 包含全面的文档和示例
5. 验证命令语法和功能
